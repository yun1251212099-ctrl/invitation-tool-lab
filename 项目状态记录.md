## 项目概述

基于 PSD 模板 + CSV/Excel 名单，批量生成个性化邀请函 PNG 图片的 Web 工具。部署在 Streamlit Community Cloud，24 小时在线，任何人浏览器打开即用。

## 技术架构

- **前端**: Streamlit (Python Web 框架)
- **PSD 解析**: psd-tools (读取图层信息)
- **图片生成**: Pillow (文字绘制、二维码替换)
- **质量检测**: OpenCV (二维码可读性检测)
- **名单解析**: pandas + openpyxl + xlrd
- **部署**: Streamlit Community Cloud (免费)
- **代码托管**: GitHub

4. 清除 `streamlit.app` 站点数据后重试

### 第5阶段: 字体精度修复 (2026-03-18)
- 问题: Pillow 和 Photoshop 渲染引擎不同, 相同字号产生不同像素大小
- 方案演变:
  1. 全局宽度校准 → 不同文字误差不一致
  2. 高度匹配 + 逐字绘制 → 间距不自然
  3. 每层独立宽度校准 → 高度差 4px
  4. **最终: 每层按高度校准, 统一 size=57, h=54px = PSD 原图**
- 绘制方式: draw.text() 自然间距, 不强制调整
- 位置对齐: 垂直中心点对齐, 0px 误差
- 预览确认机制: 必须点"效果正确"才能继续

## 已知限制 (图生图方案固有)

1. 文字渲染非 100% 像素级匹配 PS (FreeType vs Adobe 引擎)
2. 高度校准后宽度可能有 ±6px 偏差 (中文等宽字, 视觉不明显)
3. 字体笔画粗细/抗锯齿与 PS 有细微差异
4. 大文件上传到云端较慢 (Streamlit Cloud 服务器在新加坡)

如需 100% 精确匹配, 使用本地 Cursor 指令操作或 PS 脚本方案。
